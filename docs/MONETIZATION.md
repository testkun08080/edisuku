# OSS 公開 × マネタイズ戦略

作成日: 2026-06-12

---

## 基本方針

「コアは MIT で完全無料公開、価値の高い部分だけ有料にする」を軸とする。
日本の個人投資家・エンジニアが主なターゲット。

---

## 戦略 1 — ホスト版 SaaS（最優先・最大インパクト）

### 概要

自分でデプロイ・管理したくないユーザー向けに、自分が管理するインスタンスを公開する。
Cloudflare Workers の無料枠でホストできるため、低コストで始められる。

### Free / Pro 設計例

| 機能 | Free（OSS セルフホスト）| Hosted Free | Hosted Pro（月額 500〜1,000 円） |
|---|---|---|---|
| 銘柄一覧・スクリーナー | ✓ | ✓ | ✓ |
| 基本財務指標 | ✓ | ✓ | ✓ |
| データ更新頻度 | セルフで設定 | 週次 | 日次 |
| 閲覧履歴・お気に入り | ✓ | ✓（ローカル） | ✓（クラウド同期） |
| **CSVエクスポート** | ✗ | ✗ | ✓ |
| **銘柄アラート（メール）** | ✗ | ✗ | ✓ |
| **ポートフォリオ追跡** | ✗ | ✗ | ✓ |
| **過去5年以上の時系列** | ✗ | ✗ | ✓ |
| **スクリーナー条件保存** | ✗ | ✗ | ✓ |

### 実装優先度

1. まず「Hosted Free」として自分のドメインで公開（コスト: ほぼゼロ）
2. 有料機能を段階的に追加し、決済は Stripe または LemonSqueezy で実装
3. 月 10〜20 人でも収益になる規模から始める

---

## 戦略 2 — GitHub Sponsors（低コスト・今すぐ始められる）

### 概要

GitHub の Sponsors ページを有効化するだけ。
技術力を示せる OSS として公開した後、README に Sponsors バッジを置く。

### ポイント

- 月 500〜1,000 円のティアを用意（「コーヒー代」感覚）
- 特典は「Sponsor バッジ」「優先 Issue 対応」程度で十分
- 収入は少ないが**ゼロコストで始められ、信頼性の証にもなる**

---

## 戦略 3 — アフィリエイト（不労所得的）

### 証券会社アフィリエイト

財務データスクリーナーのユーザーは投資家 → 証券口座の開設につながりやすい。

候補:
- SBI証券、松井証券、楽天証券のアフィリエイトプログラム
- A8.net / バリューコマース 経由で登録可能
- 1 口座開設あたり 5,000〜10,000 円

実装: README や Hosted 版のフッターに「口座開設ボタン」を控えめに置く。

### データソース系アフィリエイト

- SPEEDA・Bloomberg 等の有料金融データサービスが紹介プログラムを持つ場合がある（要確認）

---

## 戦略 4 — 技術ブランド構築（採用・案件獲得）

直接の収益ではないが、最も持続的なリターン。

### 具体アクション

| アクション | 効果 |
|---|---|
| Zenn / Qiita に「EDINET + Cloudflare Workers で財務スクリーナーを作った」記事を書く | スター数 + 技術ブランド |
| GitHub の Star 数をポートフォリオに掲載 | フリーランス単価向上・採用 |
| Hono / Vike / Cloudflare Workers の実装事例として紹介 | 関連コミュニティへのリーチ |
| OSS をきっかけに Finance × Tech 領域のフリーランス案件を獲る | 直接収益 |

このプロジェクトは「Cloudflare Workers で動く本番品質のモノレポ」として希少性が高く、技術ブランドへの貢献度は高い。

---

## 戦略 5 — 有料テンプレ・スターターキット販売

### 概要

本リポジトリをベースに「カスタマイズ済みスターターキット」として販売。

例:
- 「EDINET スクリーナー + 銘柄アラート機能付きテンプレ」→ Gumroad / BOOTH で 3,000〜5,000 円
- 「自社データ版に差し替えられる財務 Dashboard スターター」→ エンジニア向けに販売

OSS 本体は MIT のままで、**追加機能版・セットアップ済みバンドル版**のみ有料にする形は合法で一般的。

---

## 推奨ロードマップ

### Phase 1（今すぐ・コストゼロ）

1. GitHub Sponsors を有効化し README にバッジを追加
2. Hosted 版を自分のドメインで公開（`edisuku.dev` 等）
3. Zenn に記事を書いて認知を作る
4. アフィリエイトを README / サイトフッターに設置

### Phase 2（Hosted 版が月 50〜100 人を超えたら）

5. Stripe で Pro プランの課金実装（CSVエクスポートから始める）
6. メールアラート機能を Pro 限定で実装

### Phase 3（安定してから）

7. Gumroad でテンプレ販売を試す
8. Finance × Tech のフリーランス案件に応用

## ライセンス上の注意点

現在 MIT ライセンス。MIT はコード利用・改変・商用利用を全て許可するため:

- ✓ 自分で有料 SaaS として運用できる
- ✓ 有料テンプレに含めて販売できる
- ✓ アフィリエイトリンクを付けたホスト版を運営できる
- △ 誰かがフォークして同じ SaaS を作ることも許可される（これは MIT の性質上避けられない）

「誰でもフォークできるが、自分が一番先にブランドを作った先発者」という立場を維持することが競争優位になる。

---

## まとめ

優先順位:

1. **GitHub Sponsors** — 今すぐ、ゼロコスト
2. **ホスト版公開 + アフィリエイト** — 1〜2週間で実装可能
3. **技術記事を書く** — 中長期のブランド構築
4. **Pro プラン（CSV/アラート）** — 利用者が増えてから

最大のレバレッジは「自分でホストした版を公開して認知を作ること」。
セルフホストの手間を省くだけで十分な差別化になる。
